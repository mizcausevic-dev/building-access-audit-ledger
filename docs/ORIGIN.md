# Origin

This repo exists to show that Kinetic Gain's control-plane pattern extends cleanly into proptech and real-estate access operations.

The primitive is not generic "building monitoring." The real workflow pressure lives in:

- badge approvals that need named owner proof before activation
- controller changes that need rollback notes and baseline review
- vendor access windows that must expire cleanly with the work order
- revoked credentials and audit continuity that determine whether security can trust the posture

`building-access-audit-ledger` packages that pressure into a buyer-readable and operator-readable public surface.
